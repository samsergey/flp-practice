Функторы

Типы в Haskell — это больше чем описание данных. Это, в первую очередь, описание структуры образуемой данными. Мы увидели это на примере типов для электрических сопротивлений, изображений, типов, обладающих моноидальными свойствами и т.д. Всё это относилось к способу комбинирования типов. Типы могут обладать свойствами, которые позволяют управлять потоком вычислений. Эта роль в императивном программировании отводится ветвлению, циклам или исключениям, в функциональном подходе часть и этой работы берут на себя типы.

Часто встречающийся пример — тип Maybe a, говорящий о том, что величина значения типа a может быть неопределена. Посмотрите информацию о типе Maybe с помощью команды :i. Ранее мы использовали оператор (<$>) для обработки множества значений в списке. Примените этот оператор к значениям типа Maybe. Можно ли сказать, что типы [a] и Maybe a имеют сходную структуру? Определите функции listToMaybe :: [a] -> Maybe a и maybeToList :: Maybe a -> [a]. Оператор (<$>) имеет своё имя: fmap. Выясните его тип с помощью команды :t (<$>). Посмотрите (с помощью команды :i в интерпретаторе) какие ещё знакомые вам типы принадлежат к классу Functor.

Управление сообщениями об ошибках

Тип Either похож на тип Maybe тем, что у него есть одно "верное" значение — правое, и "неверное" — левое. Но в отличие от Nothing, тип Left может нести в себе какую-либо информацию, например, сообщение об ошибке или код ошибки.

Однако тип Either не является альтернативным. Это связано с тем, что для списков и Maybe элемент empty единственен и определяется однозначно, тогда как для Either существует множество вариантов нулевых значений. Какой из них поставить в определение для empty? Но мы можем схитрить: пусть левым элементом будет какой-нибудь моноид, для него-то точно существует нулевой элемент. Ну, а раз так, то разумно будет не пренебрегать тем, что моноиды можно объединять, и использовать это свойство для объединении значений Left:

instance Monoid m => Alternative (Either m) where
  empty = Left mempty
  Left x  <|> Left y = Left (x <> y)  -- объединение нейтральных элементов дистрибутивно
  Left _  <|> x      = x               -- нейтральный элемент для объединения
  Right x <|> _      = Right x         -- нулевой элемент для объединения
Убедитесь на следующих примерах, что приведённое определение работает корректно: 
Left "fail" <|> Right 42
Right 42 <|> Left "fail"
Left "I am " <|> Left "a fail"

Теперь нужно научиться вводить в программу сообщения об ошибках, причём так, чтобы функторы Maybe и [] их игнорировали, а Either и IO использовали их. Причём, эта возможность должна быть определена для всех четырёх контекстов сразу, иначе придётся переписывать функцию sqrtA для разных контестов, а это уже не DRY (Don't Repeat Yourself — один из универсальных принципов программирования).

Раз требуется обеспечить разные типы одной функциональностью, надо определить свой класс типов. А в нём определить функцию, вводящую в нулевой элемент указанное сообщение. Назовём этот класс Failable, а функцию — message.

class (Alternative f) => Failable f where
  message :: String -> f a
  -- определение, используемое по умолчанию
  message _ = empty

и определим четыре экземляра. Для Maybe и [] используется определение функции message, данное в классе, а для Either String и IO — специфические:

instance Failable Maybe
instance Failable []
instance Failable (Either String) where message = Left
instance Failable IO where message = error
Внесите в определение функции sqrtA изменения, позволяющие использовать её в четырёх различных контекстах: 
 sqrtA 16 :: [Double]
 sqrtA 16 :: Maybe Double
 sqrtA 16 :: Either String Double
 sqrtA 16
 sqrtA (-1) :: [Double]
 sqrtA (-1) :: Maybe Double
 sqrtA (-1) :: Either String Double
 sqrtA (-1)

Нам в дальнейшей работе пригодится класс Failable, поэтому стоит выделить его и определение экземпляров Failable [], Failable Maybe Failable (Either String) и Failable IO в отдельный модуль Failable.hs

Монады

С помощью fmap можно применить любую функцию к функтору не разрушая его. А что насчёт функций, которые возвращают значение в контексте? Например, как можно дважды применить функцию sqrtM, чтобы вычислить \sqrt{\sqrt{16}}? Иными словами, как выполнить композицию функций, возвращающих не просто значение, а значение в некотором контексте m?

Попробуем начать с использования оператора <$>: 

*Main> sqrtM <$> sqrtM 16
Just (Just 2)
*Main> sqrtM <$> sqrtM (-1)
Just (Nothing)

Результат мы получили, но он оказался в контексте дважды. Какой смысл имеет выражение Just (Just 2)? С точки зрения "успешности" вычислений это просто успешное вычисление, то есть Just 2. В свою очередь выражение Just Nothing означает, что вложенное вычисление окончилось неудачей, а вместе с ним и всё вычисление.

Определите функцию join, которая занималась бы "соединением" контекстов так, чтобы можно было получить такие результаты: 
*Main> join $ sqrtM <$> sqrtM 16
Just 4
*Main> join $ sqrtM <$> sqrtM (-1)
Nothing

Связка join $ f <$> x универсальна и позволяет связывать в цепочки любые вычисления с контекстом. Для неё есть определённое в языке обозначение:

f =<< x   ≡   join $ f <$> x

Мы определили для функтора Maybe функцию join и сразу повысили её "ранг", позволяя объединять в цепочку функции возвращающие значения в контексте Maybe. Функторы для которых определено соединение join называются монадами.

Сравните типы операторов <$>, <*> и =<< и объясните своими словами их действие.

Чаще чем оператор =<< используется его аналог >>=, принимающий аргументы в обратном порядке:

(>>=) :: m a -> (a -> m b) -> m b
Вычислите следующие выражения в интерпретаторе и объясните как происходят вычисления: 
[-1,0,1,16] >>= sqrtA >>= sqrtA
Just 64 >>= sqrtA >>= sqrtA
Right 16 >>= sqrtA
Right (-16) >>= sqrtA
(read <$> getLine) >>= sqrtA >>= print

Подведём итоги.

Функтор — это тип для которого определена операция fmap (<$>).

class Functor f where
  fmap :: (a -> b) -> f a -> f b

Аппликативный функтор — это функтор, для которого определены функции pure и <*>

class Functor f => Applicative f where
  pure :: a -> f a
  <*>  :: f (a -> b) -> f a -> f b

Монада — это аппликативный функтор, для которого определена функция >>=

class Applicative f => Monad f where
   (>>=) :: Monad m => m a -> (a -> m b) -> m b

Часто для определения оператора >>= удобнее определить функцию join:

 x >>= f = join $ f <$> x
Если тип является монадой, то для полного определения его функториальных свойств достаточно определить три функции: fmap, pure и >>= (или join).
Оператор <*> выражается через >>=: 
f <*> x = f >>= (<$> x)
